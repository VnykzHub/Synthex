# artifact-factory

Generator patterns for producing YAML, Markdown, code modules, and test suites. Use this skill whenever you need to produce a structured artifact from a template-based pattern rather than writing from scratch.

## When to use
- You need to generate a YAML configuration file from a schema
- You need to produce a Markdown document with a defined structure (report, review, audit, plan)
- You need to scaffold a Python code module with __init__.py, main module, and test file
- You need to generate a test suite with coverage for a given source module
- You need to produce any artifact that follows a repeating structural pattern

## Core principles

1. **Template-first.** Always start from an existing template in `knowledgebase/templates/` before writing custom generation logic. Templates encode project conventions; custom code does not.
2. **Deterministic output.** Two invocations with the same inputs must produce identical output. Do not include timestamps or random fields unless the artifact type requires them.
3. **Idempotent structure.** Generating into an existing directory must not overwrite files that already exist unless explicitly instructed with `--overwrite`.
4. **Validation-ready.** Every generated artifact must be structurally validatable by the `structure-validator` skill. Use the correct schema, required fields, and naming conventions.
5. **Minimal surprise.** Generated files must include a header comment identifying them as auto-generated:
   - YAML: `# Auto-generated by artifact-factory`
   - Markdown: `<!-- Auto-generated by artifact-factory -->`
   - Python: `# Auto-generated by artifact-factory`

## Generator patterns

### Pattern A: YAML generator
For generating `.yaml` / `.yml` files from a schema:
1. Load the YAML schema from `knowledgebase/schemas/`.
2. Identify required top-level keys and their types.
3. Fill values from the provided parameters or sensible defaults.
4. Validate the output against the schema before writing.
5. Write with consistent 2-space indentation, no trailing spaces.

### Pattern B: Markdown generator
For generating `.md` files with a defined structure:
1. Load the Markdown template from `knowledgebase/templates/`.
2. Identify placeholder markers: `{{title}}`, `{{date}}`, `{{author}}`.
3. Replace all placeholders with provided values.
4. Fill section content from structured input (tables, lists, code blocks).
5. Verify that no unresolved placeholders remain before writing.
6. Write with consistent heading hierarchy (H1 for title, H2 for sections, H3 for subsections).

### Pattern C: Code module generator
For generating Python source modules:
1. Determine the module name and package path from the plan.
2. Generate:
   - `__init__.py` with exports
   - `<module>.py` with classes and functions (stubs or full implementation)
   - Any submodule files as needed
3. Add module docstring, class docstrings, and function docstrings.
4. Add type hints to all function signatures.
5. Generate a corresponding `tests/test_<module>.py` (see Pattern D).

### Pattern D: Test suite generator
For generating test files:
1. Read the source module to identify all public classes and functions.
2. For each public symbol, generate:
   - A test class or test function group
   - A happy-path test
   - At least one edge-case test
   - Test for expected exceptions if the function raises any
3. Structure the test file with: imports, test class (if applicable), individual test functions.
4. Each test function name must be `test_<function>_<scenario>`.
5. Add import guard: `import pytest` or `import unittest`.

## Method
1. **Analyze.** Determine the artifact type and output path. Read any relevant schema or template from `knowledgebase/`.
2. **Select pattern.** Choose the appropriate generator pattern (A-D) from above.
3. **Populate.** Fill the template with the provided parameters. For code modules, generate both source and test files.
4. **Validate.** Verify the generated artifact structurally. For YAML, validate against schema. For Markdown, check for unresolved placeholders. For code, check syntax with `python -c "import ast; ast.parse(open(path).read())"`.
5. **Write.** Write to the specified output path under the caller's writable directory.
6. **Return.** Return the list of files written.

## Output format
Return a file manifest:
```json
{
  "files": [
    {"path": "path/to/generated/file.yaml", "type": "yaml", "lines": 42},
    {"path": "path/to/generated/test_file.py", "type": "python", "lines": 89}
  ],
  "errors": []
}
```
